Change Robot Settings

The repository URL you set below should point to a git repository that is readable by the tournament runner. If it is a private repository, please contact the tournament organisers to arrange for access to be granted to the tournament runner.

The robot class name should be the name of a class within the repository that refers to your robot.

The build status indicator must be green at the time when the competition starts in order for your robot to enter the competition.
It can be red for the following reasons:

  • The repository does not exist or is not readable by the server
  • The robot class specified was not found in the repository (check your spelling)
  • The repository's pom.xml file was missing or corrupt (make sure it's checked in)
  • The repository does not produce a JAR file when the "mvn3 install" command is issued (verify locally before committing)
  • The JAR file is not placed in the correct location (the server expects it to be in the './target/' directory relative to the project root)
  • There is more than one JAR file in the './target/' directory (make sure you're not outputting a separate library or something, that will not work)
  • The build takes longer than the designated timeout (please contact the project administrators to find out what that is for this tournament); note that this is extremely unlikely unless you are actively trying to screw with the build server.
